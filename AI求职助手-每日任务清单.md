# 使用方式
每天开始前先看“当天目标”和“最小交付物”，不要贪多。
当天做完后必须截图、提交 Git 记录，并写一句当天完成了什么。
如果当天没做完，不要跳过，优先把最小交付物补齐。
# Day 1｜立项和初始化
当天目标：把前后端项目骨架搭起来，确保两边都能启动。

新建后端项目 ai-job-assistant-server，初始化 Express。
配置基础目录：前端拆出 views、components、api、stores；后端拆出 routes、controllers、services。
前端写一个最简单首页，后端写一个 /api/ping 接口。
前端请求 /api/ping，确认联调成功。
## 最小交付物：

# 每天固定检查清单

# 你每天的优先级规则
先完成主链路，再做样式优化；功能没跑通之前，不要沉迷调 UI。
先保证能演示，再考虑优雅；能跑起来比代码完美更重要。
先做必须项，再做加分项；没有数据库也能演示，但没有分析闭环就不算完成。
| 适用阶段 | 现在开始到项目第一版上线前 |
| --- | --- |
| 推进原则 | 每天只盯住当天目标，先跑通，再补全，再优化 |
| 每日投入建议 | 3–5 小时；如果当天课多，至少完成“最小交付物” |
| 交付目标 | 14 天后形成一个可演示、可写进简历、可在面试中讲清楚的项目 |
| 检查项 | 标准 |
| --- | --- |
| 页面是否可用 | 关键按钮可点击，表单不会空转 |
| 接口是否可测 | 至少能在 Postman 或页面里触发一次 |
| 状态是否完整 | 有 loading、成功、失败、空状态 |
| Git 是否提交 | 当天有明确提交记录 |
| 问题是否记录 | 写下今天卡住的点和解决思路 |
