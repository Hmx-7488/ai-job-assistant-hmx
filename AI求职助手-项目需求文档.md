AI 求职助手项目需求文档
适用对象：前端 / 偏前端全栈实习项目 · 版本 1.0


# 一、项目背景与目标
这个项目不是单纯接一个大模型 API，而是把 AI 真正嵌入求职流程，形成“上传简历 → 输入岗位 JD → 生成分析结果 → 追问和准备面试”的完整闭环。
解决用户在求职时“不知道简历哪里有问题、不知道岗位是否匹配、不知道面试怎么准备”的痛点。
用一个真实产品场景同时展示前端交互、后端接口、数据库设计、AI 集成和产品思维。
作为你的第二项目写进简历，和现有 Vue 后台项目形成互补。

# 三、第一版核心功能（MVP）
## 1. 简历上传与文本提取
支持上传 PDF 简历，后端提取原始文本。
前端展示提取结果摘要，便于用户确认内容是否正确。
第一版先支持 PDF，后续再扩展到 DOCX。
## 2. 岗位 JD 输入
用户输入岗位名称、公司名称（可选）和岗位描述文本。
保存本次分析对应的 JD，便于后续复盘。
## 3. AI 匹配分析
基于简历和 JD 输出匹配度评分、优势项、缺失项、修改建议。
要求模型按固定 JSON 结构返回，前端方便展示。
## 4. 面试题生成
根据简历和 JD 生成项目题、基础题、场景题、行为题。
支持一键复制，便于用户练习。
## 5. AI 聊天追问
用户围绕分析结果继续提问，例如自我介绍、改简历、准备面试。
保留最近几轮上下文，避免回答脱离用户背景。

# 九、Prompt 设计要求
分析 Prompt：强制要求模型输出 score、strengths、weaknesses、suggestions 四个字段，最好返回 JSON。
面试题 Prompt：要求按项目题、基础题、场景题、行为题分类输出，避免混在一起。
聊天助手 Prompt：固定角色为“务实的求职辅导助手”，优先给执行建议，不说空话。
所有 Prompt 都要在后端统一管理，避免散落在多个文件中。
# 十、开发原则
不是纯手搓到从零造轮子，而是核心链路自己做，通用能力合理借库。
先跑通 mock 数据，再接入真实 AI 接口，降低卡死风险。
先做最小闭环，再补持久化、历史记录和体验优化。
每完成一个功能就自测，保持页面状态清晰，有 loading、错误提示和空状态。
